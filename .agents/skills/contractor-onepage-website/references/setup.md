# Setup & Architecture Reference

## `astro.config.mjs`

```js
import { defineConfig } from 'astro/config';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  site: 'https://yourclient.com', // ← update before deploy
  vite: {
    plugins: [tailwindcss()],
    json: { stringify: 'auto' },
  },
});
```

**Why `json: { stringify: 'auto' }`?**
Vite 7 needs this to properly handle JSON imports inside `src/`. Without it,
large JSON files may not resolve correctly at build time.

## Tailwind v4 — CSS-first Configuration

❌ **Do NOT create `tailwind.config.mjs`** — it does not exist in v4.
✅ All config goes in `src/styles/global.css`:

```css
@import "tailwindcss";

@theme {
  --color-brand-primary: #000000;  /* from CONTENT.json company_colors.primary */
  --color-brand-accent:  #000000;  /* from CONTENT.json company_colors.accent  */
  --color-brand-light:   #f8f9fa;  /* from CONTENT.json company_colors.light   */
  --color-brand-dark:    #111827;  /* from CONTENT.json company_colors.dark    */

  --font-family-heading: 'Montserrat', system-ui, sans-serif;
  --font-family-body: 'Inter', system-ui, sans-serif;
}

@layer base {
  html { scroll-behavior: smooth; }
  body { font-family: var(--font-family-body); -webkit-font-smoothing: antialiased; }
  h1, h2, h3, h4, h5, h6 { font-family: var(--font-family-heading); }
}
```

Tailwind token classes: `bg-brand-primary`, `text-brand-accent`, `bg-brand-light`, `bg-brand-dark`

## Vite 7 — `CONTENT.json` Must Live in `src/`

Vite 7 (bundled with Astro v6) **blocks imports of files outside the `src/` boundary**.
Placing `CONTENT.json` at the project root will cause a build error.

**Correct pattern:**
```
src/
├── CONTENT.json    ← data lives here
└── content.ts      ← thin re-export module
```

`src/content.ts`:
```ts
import content from './CONTENT.json' with { type: 'json' };
export default content;
```

Every component:
```ts
import content from '../../content.ts';
```

## Image Pipeline

```
COMPANY-PHOTOS/ (raw)
      ↓ node scripts/process-images.mjs
src/assets/images/ (cropped + resized, still high quality JPG)
      ↓ Astro <Image /> at build time
dist/ (WebP, compressed, with srcset)
```

**Aspect ratios by section:**

| Section | Ratio | Max Width |
|---------|-------|-----------|
| hero | 16:9 | 1920px |
| welcome | 4:3 | 800px |
| services | 4:3 | 600px |
| about | 4:3 | 800px |
| gallery | 1:1 | 800px |
| logo | as-is | — |

**Favicon:** 50×50 PNG, `fit: 'contain'`, transparent background.
Never use `fit: 'cover'` — portrait logos will be cropped.

## Logo Sizing in Navbar

Astro's `<Image />` requires explicit `width` and `height`.
Calculate `width` from the actual logo pixel dimensions:

```
logo is 300×319px, desired height = 80px
width = Math.round(300 * (80 / 319)) = 75px
```

## Netlify Forms

Both `LeadForm.astro` and `ContactUs.astro` must have:
```html
<form name="[unique-name]" method="POST" data-netlify="true" action="/thank-you">
  <input type="hidden" name="form-name" value="[unique-name]" />
</form>
```

`action="/thank-you"` is mandatory. Without it Netlify shows its own generic confirmation page.

## Design Tokens

| Token | CSS Variable | Usage |
|-------|-------------|-------|
| `bg-brand-primary` | `--color-brand-primary` | CTA buttons, active nav links |
| `text-brand-accent` | `--color-brand-accent` | Hover states, icons, badges |
| `bg-brand-light` | `--color-brand-light` | Alternate section backgrounds |
| `bg-brand-dark` | `--color-brand-dark` | TopBar, Footer, dark sections |

## Spacing System

- Section padding: `py-20` (desktop) / `py-12` (mobile)
- Container: `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
- Sections alternate: `bg-white` / `bg-brand-light`

## Naming Conventions

| Type | Convention | Example |
|------|-----------|---------|
| Components | PascalCase | `ServiceCard.astro` |
| Pages | kebab-case | `thank-you.astro` |
| Images | kebab-case | `interior-painting.jpg` |
| Section IDs | lowercase hyphen | `id="why-choose-us"` |
| Props | camelCase | `serviceName` |
